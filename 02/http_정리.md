## 1. HTTP 내용 정리

### 1-1. HTTP

- HyperText Transfer Protocol의 줄임말로, 글 또는 그림에 링크를 넣어 다른 정보로 쉽게 이동할 수 있는 텍스트를 전송하기 위해 사용되는 통신 규약이다.
- HTTP를 사용하는 이유는 클라이언트와 웹 서버가 통신을 할 때 표준 방법이 필요하기 때문이다.

### 1-2. HTTP Message

- 클라이언트와 서버 간 파일 등을 주고 받는 통신의 기본 단위
- 클라이언트와 서버 간 데이터를 교환하기 위해서 사용한다.

#### HTTP Message의 구성요소

| 순서  |        이름        | 내용                                                      |
| :-: | :--------------: | :------------------------------------------------------ |
|  1  | 시작 줄(Start Line) | 요청이나 응답의 상태를 나타내는 첫 번째 줄                                |
|  2  | 헤더(HTTP Headers) | 메시지 바디를 요약하는 헤더들의 집합                                    |
|  3  | 빈 줄(Empty Line)  | 헤더와 본문을 구분하기 위해 존재하는 빈 줄                                |
|  4  |     본문(Body)     | 데이터나 문서 등 실제 내용이 포함되며 HTML 문서, JSON 데이터 등이 본문에 담길 수 있다. |

### 1-3. HTTP Request Method

- 웹 서버에게 어떤 작업을 수행하길 원하는 지 알리는 방법을 정의하는 명령 규칙
- 웹 서버에게 구체적인 작업을 명확하게 지시하기 위해서 사용한다.

|  메서드   | 특징                                          |
| :----: | :------------------------------------------ |
|  GET   | 서버에게 데이터 조회를 요청할 때 사용한다.                    |
|  POST  | 서버로 데이터를 전송하고, 리소스를 생성하거나 업데이트 하기 위해서 사용한다. |
|  PUT   | 서버로 데이터를 전송해 리소스 전체를 업데이트 하기 위해서 사용한다.      |
| PATCH  | 서버의 리소스를 부분적으로 수정 또는 업데이트 하기 위해서 사용한다.      |
| DELETE | 서버로부터 리소스를 삭제하기 위해서 사용한다.                   |

### 1-4. HTTP - Status Code

- 웹 통신 규약에서 요청의 처리 상태를 나타내는 코드
- 클라이언트가 웹 서버에 요청을 했을 때 통신 결과를 웹 서버가 클라이언트에게 명확하게 전달하기 위해 사용한다.

| 번호대 | 설명                      |
| --- | ----------------------- |
| 1xx | 정보 메시지 (Informational)  |
| 2xx | 성공 (Successful)         |
| 3xx | 리다이렉션 (Redirection)     |
| 4xx | 클라이언트 오류 (Client Error) |
| 5xx | 서버 오류 (Server Error)    |

### 1-5. HTTP - URL

- 리소스의 위치와 종류를 나타내는 주소
- 웹 상의 리소스를 식별하고 접근하기 위해서 사용한다.

``` 
http://www.example.kr:80/path/to/example.html?key1=value1&key2=value
```

위와 같은 URL이 있을 때 각 부분은 다음과 같다.

| 내용                     | 구성 요소   | 의미                                                                                                          |
| ---------------------- | ------- | ----------------------------------------------------------------------------------------------------------- |
| http                   | 스키마     | 브라우저가 리소스를 요청할 때 사용해야 하는 프로토콜.                                                                              |
| www.example.kr:80      | 도메인과 포트 | 도메인은 요청하는 웹 서버, 포트는 웹 서버의 리소스에 접근하는데 사용되는 "게이트".<br>HTTP 프로토콜의 표준 포트(HTTP: 80, HTTPS: 443)를 사용하는 경우에는 생략한다. |
| path/to/example.html   | 리소스 경로  | 웹 서버에 있는 리소스의 경로.                                                                                           |
| key1=value1&key2=value | 매개변수    | 웹 서버에 제공하는 추가 매개변수.<br>'?' 뒤에 작성하고 '&'을 사용해 매개변수를 구분한다.                                                     |
