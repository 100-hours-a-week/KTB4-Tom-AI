## 회고

### 배운 점
- 여러 알고리즘을 직접 사용해보며 각 알고리즘이 어떤 특징을 가지고 있는지 배울 수 있었다.
- 클래스 불균형이 심한 상황에서 데이터 증강을 했을 때 증강 전 후 성능 비교를 정확도만으로 하면 안 된다는 것을 배웠다. 클래스 불균형이 심하면 한 클래스로만 예측을 해도 높은 정확도를 달성할 수 있기에 f1 스코어와 같은 다른 성능 평가 지표를 사용해야 한다.
- pytorch로 직접 모델을 설계하며 어떤 식으로 구성해야 하는지, 학습과 평가는 어떻게 해야하는지 알게 되었다.

### 어려웠던 점
- 활성화 함수를 직접 만들 때 어떻게 만들어야 좋은 활성화 함수가 되는지 어려웠다.
- pytorch로 모델을 직접 설계하는건 오랜만에 해봤다. 그래서 pytorch의 여러 문법들에 익숙해지고 어떤 기능을 하는 것인지 찾아보는데 시간이 좀 걸렸다.

### 보완하고 싶은 점
- 활성화 함수를 만들었는데, 회고하면서 생각하니 왜 활성화 함수를 이렇게 만들었는지에 대한 깊은 생각이 있지 않았던 것 같다. 다시 내가 만든 활성화 함수를 보며 왜 이렇게 만들었고, 좋은 활성화 함수는 무엇인지 찾아보며 수정을 하고 싶다.
- pytorch를 사용해 모델을 만들었지만 완벽한 이해 하에 완성된 것이 아니라 생각하기 때문에 다시 복습을 하고 코드를 뜯어보며 이해하는 시간을 가지고 싶다.
- 설계한 모델들의 각 층에 대한 하이퍼파라미터를 많이 건드려보지 못한 것 같다. 복습할 때 하이퍼파라미터를 바꿔가며 어떤 차이점이 있는지 알아가려 한다.

### 최종 회고
- 지금까지 공부를 해오며 pytorch를 이용한 모델 설계를 할 일이 별로 없었다. 그래서 이렇게 직접 설계할 수 있는 시간이 있어서 매우 뜻깊었다. 모델을 이렇게 직접 만들어보고 각 계층에 대한 깊은 이해를 갖는 것이 중요하다고 생각해서 이번 한 번만으로 끝내지 않고 복습을 계속 할 예정이다. 
- 결과를 시각화하는 코드 부분들은 AI의 도움을 받아 작성했고, 단순히 복붙을 하는 것이 아니라 코드를 따라치며 각 코드가 어떤 기능을 하는지 최대한 알기 위해 노력했다.
- 강사님이 보여주신 회고를 참고해서 작성해봤는데 메타인지에 아주 좋은 것 같다. 앞으로도 내가 한 것을 이러한 형식으로 작성하며 정리하려고 한다.
